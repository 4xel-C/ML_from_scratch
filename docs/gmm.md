# GMM — Gaussian Mixture Model

**Type:** Unsupervised — Clustering (soft assignment)  
**File:** `clustering/gmm.py`  
**Probabilistic counterpart of:** K-Means

---

## Problem

K-Means assigns each point to exactly one cluster (hard assignment). GMM generalizes this: each cluster is a **multivariate Gaussian**, and each point gets a **probability of belonging** to every cluster (soft assignment).

---

## Model

The data is assumed to be generated by a mixture of K Gaussians:

```
p(x) = sum_k  pi_k * N(x | mu_k, Sigma_k)
```

Each component k is parameterized by:
- **mu_k** (K, p) — centroid (mean of the Gaussian)
- **Sigma_k** (K, p, p) — covariance matrix (shape and spread)
- **pi_k** (K,) — mixing weight (proportion of points in cluster k)

Constraints: `sum(pi_k) = 1` and `0 <= pi_k <= 1`.

---

## Multivariate Gaussian

```
N(x | mu, Sigma) = 1 / sqrt((2*pi)^p * det(Sigma)) * exp(-0.5 * (x - mu)^T Sigma^{-1} (x - mu))
```

The term `(x - mu)^T Sigma^{-1} (x - mu)` is the **Mahalanobis distance** — it measures how far x is from mu, accounting for the shape of the distribution.

---

## EM Algorithm (Expectation-Maximization)

### Initialization
- **mu_k**: K random points from X
- **Sigma_k**: identity matrix I_p for each cluster
- **pi_k**: 1/K (uniform)

### E-step — Compute responsibilities

The responsibility r(i, k) is the posterior probability that cluster k generated point x_i:

```
r(i, k) = pi_k * N(x_i | mu_k, Sigma_k) / sum_j(pi_j * N(x_i | mu_j, Sigma_j))
```

This is Bayes' rule: prior * likelihood / evidence.

Shape of r: **(n, K)** — each row sums to 1.

### M-step — Update parameters

Using the responsibilities as soft assignments:

**Centroids** (weighted mean):
```
mu_k = sum_i(r(i,k) * x_i) / sum_i(r(i,k))
```

**Covariance matrices** (weighted outer product):
```
Sigma_k = sum_i(r(i,k) * (x_i - mu_k)(x_i - mu_k)^T) / sum_i(r(i,k))
```

**Mixing weights** (effective proportion):
```
pi_k = sum_i(r(i,k)) / n
```

### Convergence

Stop when the log-likelihood plateaus:
```
L = sum_i log( sum_k pi_k * N(x_i | mu_k, Sigma_k) )
```

Condition: `|L_new - L_old| < tol` or max iterations reached.

---

## Vectorized Mahalanobis Distance

For each cluster k, compute the Mahalanobis distance for all n points at once:

```python
diff = X - centroids[k]                                              # (n, p)
mahalanobis = np.sum(diff @ np.linalg.inv(covariances[k]) * diff, axis=1)  # (n,)
```

Breakdown:
- `diff @ inv(Sigma_k)` → (n, p) @ (p, p) → (n, p)
- element-wise `* diff` → (n, p)
- `sum(axis=1)` → (n,)

This avoids a loop over n points — only one loop over K clusters remains.

---

## Vectorized Covariance Update

Using the square-root trick to avoid an explicit loop over points:

```python
X_weighted = sqrt(r[:, k])[:, np.newaxis] * (X - centroids[k])   # (n, p)
Sigma_k = (X_weighted.T @ X_weighted) / sum(r[:, k])              # (p, p)
```

This works because: `X_weighted.T @ X_weighted = sum_i(r(i,k) * (x_i - mu_k)(x_i - mu_k)^T)`.

---

## K-Means vs GMM

| | K-Means | GMM |
|---|---|---|
| Assignment | Hard (0 or 1) | Soft (probabilities) |
| Cluster shape | Spherical | Arbitrary (via covariance) |
| Parameters per cluster | centroid only | mu, Sigma, pi |
| Algorithm | EM (simplified) | EM (full) |
| Output | cluster labels | labels + probabilities |

K-Means is a special case of GMM where all covariances are `sigma^2 * I` and responsibilities are binary.

---

## Complexity

- E-step: O(n * K * p^2) — one matrix inverse per cluster, applied to all points
- M-step: O(n * K * p^2) — covariance update dominates
- Total: O(iterations * n * K * p^2)
- Matrix inverse `np.linalg.inv` is O(p^3) per cluster per iteration
